2. que cambia vs el interprete base

- ya no existe let ni letrec

antes todo se hacia con scopes tipo:
let x = ...
letrec f = ...

aca no.
ahora var y const viven como expresiones con side effects.

eso significa:
cuando hago

var x = 42

no solo devuelve 42, tambien modifica el env actual y deja x guardada.

o sea eval ya no puede devolver solo valor.
toca devolver algo tipo:

(valor, env_actualizado)

y cada expresion siguiente usa ese env nuevo.

ej:

var x = 42
var y = x + 1

x tiene que existir cuando y se evalua.

---

funciones

antes recursion = letrec

ahora TODA func es recursiva por default.

si hago:

func fib(n) { ... }

fib ya tiene que existir dentro de su propio cuerpo.

solucion:
usar box.

flujo:
1. crear caja vacia
2. crear closure apuntando al env con esa caja
3. meter closure en la caja

asi fib puede llamarse a si misma.


---

parametros

mezcla rara:

por valor:
- nums
- strings
- bools
- funcs

por referencia:
- listas
- diccionarios

basicamente si el arg es mutable, pasar referencia.
si no, copia.


---

control flow nuevo

while
while cond do body done

loop clasico.
retorna null.

for
for x in lista do body done

itera lista.
cada vuelta x apunta al elemento actual.

switch

switch x {
 case ...
 default ...
}

comparacion estructural.
agarra el primer match.

print(exp)

imprime y retorna null.


---

secuenciacion

importantisimo.

si hago:

x = x + 1;
print(x);

cada expresion tiene que pasarle el env actualizado a la siguiente.

si no haces eso todo explota.
